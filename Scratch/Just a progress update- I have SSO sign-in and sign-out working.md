Another a progress update:  I have SSO sign-in and sign-out working on both sites.  However, during test, I discovered an issue with Safari.  After doing some research, I discovered that Safari is the only browser that doesn't allow cookies to be set inside iframes.  This includes local storage, which is the basis for the SSO functionality.  I spent the majority of yesterday building a workaround for this, which will require an explicit redirect from one website to the other, then back again, in the main window.  I've build some blank pages for the system to land on, so that the redirect is minimally intrusive/distracting.  We could consider add a simple design and some info on these pages about logging in.
